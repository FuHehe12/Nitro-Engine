# Refinement Guide

> Optional step. Read and execute only when the user explicitly requests it.
> Sub-agents handle the actual refinement; the primary agent oversees the process and makes the final call.

---

## When to Run Refinement

Refinement is not a default step — **it is triggered only when the user explicitly requests it** (e.g., "refine," "polish," "fine-tune," "fix formatting"), and the primary agent must not enable it on its own judgment.

Refinement consists of two steps, each independent — they can be run individually or together:

| Step | Example trigger words | What it does |
|------|-----------------------|--------------|
| A: Format repair | "fix formatting," "clean up tags," "fix headings" | Repairs format defects that require semantic understanding |
| B: Content refinement | "polish," "fine-tune," "remove translationese" | Improves the prose quality of the translation |

---

## Step A: Format Repair

Repairs format issues that scripts cannot handle — those requiring semantic understanding.

### Typical Issues

**Markdown generated by tools such as MinerU**:
- **Scrambled heading hierarchy**: Body paragraphs or table-of-contents entries are incorrectly tagged as `#`/`##`/`###` headings; correct the hierarchy according to document logic
- **Heading-level tags mixed into table-of-contents sections**: TOC list items are recognized as headings, fragmenting the document structure; restore them as ordinary list items

**Markdown converted from HTML**:
- **Residual HTML tags**: Tables retained as `<table><tr><td>` → convert to standard Markdown tables
- **Inline style tags**: `<span style="...">` / `<font>` / `<br>` → remove or replace with Markdown equivalents
- **Image and link tags**: `<img src="...">` → `![](...)` format

**Formula formatting**:
- LaTeX formulas should use `$...$` (inline) or `$$...$$` (block); other formats (bare LaTeX, MathML) must be converted to this standard

### Sub-Agent Format Repair Prompt Template

```
You are a Markdown format repair specialist. Fix the formatting issues in the translation without altering any text content.

Your inputs:
1. Translation file: [path to translation]

Repair checklist:
1. **Heading hierarchy**: TOC entries incorrectly tagged as headings → restore as ordinary list items; heading levels that skip in the body → correct according to document logic
2. **HTML tag cleanup**: Residual `<table>`/`<span>`/`<font>`/`<img>` tags → convert to standard Markdown
3. **Formula formatting**: Non-standard formulas → normalize to `$...$` (inline) or `$$...$$` (block)
4. **Basic formatting**: Unclosed code blocks → close them; misaligned list indentation → correct; incomplete table formatting → complete

Principle: fix formatting only, do not alter text. If a structural issue cannot be confidently resolved, leave it as-is and add a comment at the end of the file.

Output: Edit the file in place (use the Edit tool). Do not create new files.
```

---

## Step B: Content Refinement

Improves the prose quality of the translation and eliminates translationese.

### Sub-Agent Content Refinement Prompt Template

```
You are a [target language] writing editor (target region: [target region]). Give the translation a second pass to eliminate translationese and improve overall prose quality.

Your inputs:
1. Translation file: [path to translation]
2. Glossary: [path to _glossary.md]
3. Translation brief: [path to _translation_brief.md]

Refinement checklist — assume the reader has already said "it still reads like a translation," and go through each paragraph with that assumption in mind:

1. **Translationese scan**: Identify constructions such as "被……所……", Europeanized phrasing, and awkward structures; rewrite them as natural expressions in the target language
2. **Rhythm check**: Break up runs of more than three long sentences; consolidate places where short sentences are stacked unnecessarily
3. **Terminology consistency**: Search the full text for key terms and confirm the chosen translations are used consistently throughout
4. **Regional style check**: Confirm that word choices fit the conventions of the target region
5. **Cut the filler**: Remove empty words such as "进行", "一个", "相关的" that carry no information

Refinement principle: subtract, don't add. Like a sculptor removing excess stone — let the translation reveal its natural shape.

Output: Edit the file in place (use the Edit tool). Do not create new files.
```

## Sub-Agent Configuration

Both steps share the same configuration:
- `subagent_type`: "general-purpose"
- `model`: "sonnet"
- Before dispatching, replace all variables in the template with actual values:
  - `[target language]` → defaults to "Chinese"; fill in based on the translation brief
  - `[target region]` → defaults to "Mainland China"; fill in based on the translation brief

## Scale Strategy

The primary agent determines refinement granularity based on file size (applies to both Step A and Step B):

| Scale | Strategy |
|-------|----------|
| Small file (≤ 2500 lines) | Dispatch 1 sub-agent to process the full text |
| Large file (> 2500 lines) | Dispatch sub-agents to process segment seams + spot-check 2–3 segments |
| Multiple files | Dispatch 1 sub-agent per file for spot-check processing |

## Primary Agent Control Flow

1. Ask the user whether format repair is needed (Step A)
2. If yes → dispatch sub-agent → review output
3. Ask the user whether content refinement is needed (Step B)
4. If yes → dispatch sub-agent → review output
5. Report a summary of the refinement to the user
